# Immich AutoTag v0.80.0 – User Groups, Smarter Rule Engine, & Advanced Album Automation!

Hi Immich community!

I'm excited to announce **v0.80.0** of **Immich AutoTag** — a major milestone that consolidates months of development in user permissions, automatic album creation, and rule engine logic. This release marks the beginning of our stabilization roadmap toward **v1.0.0**, tested against libraries of over **360,000 assets** to ensure scalability.

## What's new in v0.80.0?

* **Smart Album Creation & Assignment:** Automatically create and manage albums based on your filesystem's folder structure or detected duplicates within Immich. A massive time-saver for large-scale library organization.
* **Rule-Based Classification Engine:** Define flexible rules using tags and album name patterns. The tool automatically categorizes assets, flagging them as *classified*, *in conflict*, or *pending* based on your custom logic.
* **Automated Permission Management:** Group users and define rules in your config file to automatically assign album permissions based on keywords in album names—perfect for managing access across large family archives.
* **Proactive Date Correction:** When enabled, the tool fixes asset dates by analyzing duplicates or extracting metadata from filenames (including common Android and WhatsApp naming patterns).
* **Enhanced Reliability:** Refined logic to prevent "orphaned" media and ensure library integrity.
* etc

## ⚠️ Known Issue — Docker

In some environments, the **Docker image** (`0.80.3`) may experience connectivity issues with the Immich API (likely DNS/network related). If you encounter this, we recommend using the **pipx** method as a temporary workaround:

```bash
pipx run immich-autotag
```

Tracked at [GitHub Issue #43](https://github.com/txemi/immich-autotag/issues/43). Fix in progress.
